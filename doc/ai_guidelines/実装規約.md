# 実装規約（Human + Agent）

> 最終更新: 2026-06-19

## 1. 変更前確認

- 実装前に `doc/specs/02_要件定義.md` との差分を確認する
- 要件外の機能は実装せず、先に要件更新を提案する

## 2. PR 粒度

- 1 PR = 1 責務（目安 15〜300 行）
- コミットメッセージは日本語 + Conventional Commits 接頭辞

## 3. 命名規約

| 対象 | 規則 |
|---|---|
| 変数・関数 | `snake_case` |
| クラス | `PascalCase` |
| 定数 | `UPPER_SNAKE_CASE` |

## 4. テスト規約

- 新規ロジックには最低 1 件の unit test を追加
- `tests/unit/` では外部 API 禁止
- ノートブックの検証ロジックは src へ移したタイミングでテスト化

```bash
uv run pytest tests/unit -v
uv run pytest tests/integration -v
```

## 5. データ分析の運用

- 乱数 seed は定数化し、再現可能にする（`config/.env` またはコード内定数）
- 前処理パイプラインは `src/analysis_titanic/` に集約する
- ノートブックは探索用。確定した処理はパッケージへ移す

## 6. セキュリティ

- 秘密情報をコード・ノートブック・Git に含めない
- `config/.env` は Git 除外（`.env.example` のみコミット）
